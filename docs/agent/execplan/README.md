# ExecPlan Guide

PatchPilot implementation work should use execution plans under:

```text
docs/plans
```

Each plan is a living document. Keep these sections updated:

- Purpose / Big Picture
- Progress
- Surprises & Discoveries
- Decision Log
- Outcomes & Retrospective
- Context and Orientation
- Plan of Work
- Concrete Steps
- Validation and Acceptance
- Idempotence and Recovery
- Artifacts and Notes
- Interfaces and Dependencies

Plans should be detailed enough that another engineer can continue the work without guessing intent.
